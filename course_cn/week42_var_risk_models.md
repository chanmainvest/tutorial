# 第四十二周：风险价值与风险模型——参数法、历史法、蒙特卡洛法，以及为何三种方法在尾部均告失效

## 第一部分：阅读章节

### 1. 为何本节至关重要

上周我们搭建了风险管理的概念框架：仓位管理、止损规则、情景分析，以及如何在各持仓之间分配投资组合的痛点预算。本周我们将为这些概念赋予具体数字。

**风险价值（VaR）** 是全球引用最广泛的单一风险指标。每一家银行、对冲基金、养老金机构和保险公司的资产负债表管理体系中，某个环节都在运行 VaR 系统。**CVaR / 预期损失（Expected Shortfall）** 是其更为成熟的变体——自2016年《巴塞尔协议III》起被强制采用，原因正是普通 VaR 在2008年的惨败。两者源于同一理念：构建投资组合可能结果的概率分布，审视最糟糕情形有多惨，然后用单一美元数值加以表达。

理解这些指标有四个理由。

1. **共同语言。** 当交易对手说"我们的单日99% VaR 为四千万"，这是一个精确且可证伪的声明。如果你无法将其转译为"他们预期每一百个交易日中有一天的损失超过四千万美元，而对最坏情形究竟有多糟，他们一无所知"，你就无法评估这一声明。
2. **三种计算方法，三套谎言。** 参数法 VaR 假设收益率服从正态分布——这是错的，尤其在尾部。历史法 VaR 对历史数据重新抽样——它只能反映你恰好持有的那段历史。蒙特卡洛法 VaR 要求你指定一个模型——而你的模型不过是一个猜测。三种方法在最关键的地方分歧最大：99% 和 99.9% 分位数处，以及每十年最惨的那一周。
3. **肥尾问题。** 自1990年以来，美国股票日收益率在各个五年滚动窗口中的峰度介于7至15之间，从未接近正态分布的理论值3。参数法 VaR 在95%置信水平下对实际损失的低估幅度约为10%至30%。在99%置信水平下，低估幅度为2至5倍。在99.9%置信水平下，这个数字基本上是科幻小说的内容。
4. **CVaR 是监管机构目前使用的指标，也是你应该使用的。** 巴塞尔委员会在2016年《交易账簿基本审查》中用预期损失取代了 VaR，正是因为 CVaR 对整个尾部求平均，而非仅读取分布上的某一点。它更为保守，更难被操控，在数学意义上也比 VaR 更具一致性。

本课最后的核心结论：*参数法 VaR 在95%置信水平下尚可接受，在99.9%置信水平下毫无意义，而这两者之间的差距，在1998年吞噬了长期资本管理公司，在2008年吞噬了结构性信贷部门，又在2020年3月吞噬了杠杆波动性目标基金。*

### 2. 你需要掌握的内容

#### 2.1 定义——一句话，三个参数

风险价值是在**给定概率和给定时间跨度内不会被超过的最大损失**。三个参数，缺一不可：

1. **置信水平** $\alpha$。常用取值：95%、99%、99.5%、99.9%。其补集 $1-\alpha$ 是我们度量的尾部概率——99%置信水平对应的是最差1%的结果。
2. **时间跨度** $T$。常用取值：1天、10天、1个月、1年。银行交易部门使用1天。巴塞尔标准使用10天。资产配置者使用1个月或1年。
3. **货币金额。** VaR 数值本身，以美元计。

教科书式的表述：

> *单日99% VaR = \$5,000,000* 意味着：在每100个交易日中的99天里，我们预期投资组合单日损失不超过五百万美元。在剩余的1天，损失可能更大——而 VaR 对损失究竟有多大只字不提。

最后这一句话才是关键。VaR 告诉你的是损失超过 $1-\alpha$ 概率的*阈值*。它对突破阈值时损失有多惨，没有任何描述。两个投资组合可以拥有完全相同的99% VaR，却有截然不同的99.9% VaR——一个上限为-\$10m，另一个无界。

#### 2.2 方法一——参数法（方差-协方差法）

最快、最古老、也最容易出错的方法。假设日收益率服从均值为 $\mu$、标准差为 $\sigma$ 的正态分布，则置信水平 $\alpha$、时间跨度 $T$ 下的 VaR 为：

$$ \text{VaR}_\alpha = -\big(\mu T - z_\alpha \,\sigma\sqrt{T}\big)\,V $$

其中 $V$ 为投资组合价值，$z_\alpha$ 为标准正态分位数。经典 $z$ 值如下：

| $\alpha$ | $z_\alpha$ | 尾部概率 |
|---|---:|---:|
| 90% | 1.282 | 10% |
| 95% | 1.645 | 5% |
| 97.5% | 1.960 | 2.5% |
| 99% | 2.326 | 1% |
| 99.5% | 2.576 | 0.5% |
| 99.9% | 3.090 | 0.1% |
| 99.99% | 3.719 | 0.01% |

代入标普500日度数据：$\mu \approx 0.04\%$（每日），$\sigma \approx 1.1\%$（每日）。以\$1,000,000投资组合为例：

- 单日95% VaR：$-(0.0004 - 1.645 \times 0.011) \times 1{,}000{,}000 \approx \$17{,}700$。
- 单日99% VaR：$\approx \$25{,}200$。
- 单日99.9% VaR：$\approx \$33{,}600$。

这些是1990年代银行电子表格的输出结果。再看看现实：1987年10月19日，标普500单日暴跌 **20.5%**。在参数模型下，这是一个 $z = 19$ 的事件——其概率为 $10^{-79}$，意味着*从现在到宇宙热寂都不应该发生*。然而它发生了。就在一个星期一。里根还在白宫。

这一个数据点就足以说明模型是错的。

参数法 VaR 的**优势**：快速、可加性强，易于在数千个持仓中传导计算。方差可加，相关性是线性的，可以在电子表格中完成。

参数法 VaR 的**劣势**：$\sigma$ 在平静时期的窗口中计算，却被用于预测动荡时期；正态分布的尾部极薄，任何真实投资组合都从未经历过这样的尾部。*在95%置信水平下，它是一个估计。在99%置信水平下，它是一个猜测。在99.9%置信水平下，它是虚构。*

#### 2.3 方法二——历史模拟法

完全摒弃正态假设。取过去 $N$ 天的投资组合收益率数据（通常为250至1,000个交易日），排序后直接读取经验 $1-\alpha$ 分位数。若 $N=1000$，$\alpha=99\%$，则99% VaR 即为样本中第10个最差收益率。

实例演示。取 SPY 过去1,000个交易日数据并排序，第10个最差值约为 $-3.6\%$。对 \$1m 投资组合而言，单日99%历史法 VaR 为 \$36,000——比上述参数法结果**约高40%**。历史数据"记得"新冠疫情，记得2018年第四季度的波动，且每天的数据权重相同。

**优势**：无需参数假设。如果历史存在肥尾，VaR 自然就有肥尾。

**劣势**：你只能看到已经发生的事情。如果你的1,000天窗口截止于2007年10月，数据中便没有2008年的内容，一旦市场机制骤变，你的99% VaR 读起来就像95%的水平。窗口太短（不稳定，无尾部），太长又会混入已不适用的旧机制。

部分补救措施包括：**年龄加权自助法**（近期数据权重更高）或**过滤历史模拟法**（将昨日收益率重新缩放至今日波动性水平）。两者都使模型更贴近实操中看图的直觉。

#### 2.4 方法三——蒙特卡洛模拟法

指定一个模型。从中模拟 $M$ 条路径。计算每条路径下的投资组合盈亏。排序。读取分位数。

模型可以是正态分布（此时蒙特卡洛结果与参数法一致，仅有抽样噪声）。也可以是自由度较低的 Student-t 分布（尾部更厚）。或者是机制转换混合模型（平静日来自一个正态分布，动荡日来自另一个）。或者是 GARCH 模型（今日波动率取决于昨日波动率）。或者是跳跃扩散模型（正态噪声加偶发性泊松跳跃）。

灵活性是完全的。当前美国股票投资组合日度蒙特卡洛的"现代"标准通常采用自由度为5至7的 Student-t 分布——产生与经验分布近似匹配的尾部。这是2008年后银行自营部门普遍采用的模型。

**优势**：可捕捉非线性效应（期权、结构化产品）、路径依赖型收益（障碍产品、可赎回债券），以及任何可以写下来的非正态分布。

**劣势**：*垃圾进，垃圾出*。如果你的模型在尾部有误，蒙特卡洛 VaR 在尾部也有误——还带着精确性的幻觉。用错误模型跑100,000条路径，只会给你一个信心满满的错误答案。

诚实的从业者会同时运行三种方法，将三者之间的差距视为数字中不可消除的不确定性，并报告其中最为保守（最大）的那个。

#### 2.5 CVaR / 预期损失——尾部的平均值

CVaR（又称预期损失，ES）回答了 VaR 所回避的问题：*既然损失已经超过 VaR 阈值，平均损失是多少？*

$$ \text{CVaR}_\alpha = \mathbb{E}[L \mid L \geq \text{VaR}_\alpha] $$

对于正态分布和连续损失 $L$，CVaR 有封闭解：

$$ \text{CVaR}_\alpha = \mu + \sigma\,\frac{\phi(z_\alpha)}{1-\alpha} $$

其中 $\phi$ 为正态分布的概率密度函数。在正态分布假设下，$\text{CVaR}/\text{VaR}$ 比率在95%置信水平下约为 **1.25**，在99%置信水平下约为 **1.15**——意味着在正态假设下，平均突破损失仅比 VaR 阈值大15%至25%。然而在美国股票市场的实证数据中，这一比率接近 **1.30至1.50**，尤其在99%以上的置信水平。尾部不仅比正态分布预测的更厚；突破阈值时的*深度*同样更大。

CVaR 具有**一致性**（正式意义上，即1999年 Artzner-Delbaen-Eber-Heath 论文框架）——满足任何合理风险度量应具备的四个性质：单调性、次可加性、正齐次性和平移不变性。**VaR 不具备一致性**——对于重尾分布，VaR 会违反次可加性，即两个投资组合合并后的 VaR 可能*大于*各自 VaR 之和。这一数学病态在实践中有深刻影响：VaR 可能激励隐性尾部风险集中，而 CVaR 对此有惩罚机制。巴塞尔协议III正是出于这一原因，在97.5%置信水平上以预期损失取代了 VaR。

#### 2.6 为何三种方法在尾部均告失效

根本问题可以用一个数字概括：**峰度**。

正态分布的峰度恰好为3（即超额峰度为零）。峰度越高意味着尾部越厚——正态分布允许范围之外（±3σ以外）的概率质量越多。

数据怎么说？对1990年以来标普500日收益率计算滚动五年峰度，答案是：*从未接近3*。通常介于7至15之间。包含1987年的窗口峰度超过30。包含2008年的窗口约为12。即便是2003至2007年这样平静的窗口也约为5。

![标普500日度收益率滚动五年峰度折线图，时间跨度为1990年至2026年4月。折线在约4至30之间波动，在包含1987年、2008至2009年、2010年闪崩以及2020年3月的窗口处出现显著峰值。水平虚线标注峰度=3，即正态分布基准。经验折线在样本的每一天都高于3，大部分时间介于7至12之间。](../image/week42_kurtosis_history.png)

对 VaR 的影响：

- **95%置信水平下：** 参数法 VaR 低估幅度为5%至15%。可接受。
- **99%置信水平下：** 参数法 VaR 低估幅度为30%至100%。实质性偏差。
- **99.9%置信水平下：** 参数法 VaR 低估幅度为 **2至5倍**。毫无意义。
- **99.99%置信水平下：** 参数法 VaR 是礼貌的虚构。

1998年的长期资本管理公司爆仓，在该基金模型的视角下并非百万分之一的事件；回头来看，这不过是一个五十年一遇、模型却无法识别的情景。2008年的抵押贷款债券损失，据高盛时任首席财务官 David Viniar 所言，是"连续数天出现二十五倍标准差事件"——这是对模型形状根本错误的坦白，而非世界本身出了问题。2020年3月，SPY 单日交易跌幅达12%，而参数模型给出的概率约为 $10^{-25}$。

正确的教训不是"用更复杂的 VaR"。正确的教训是：*风险数字都是估计值，全都如此，而尾部是估计值撒谎最厉害的地方*。尾部驱动全局。你的风险系统必须包含在定量 VaR 若干倍水平上的明确压力测试，无论模型声称什么是"不可能的"。

#### 2.7 综合运用——实用框架

严肃的投资者如何使用这些工具：

1. **同时运行三种方法。** 参数法、历史模拟法、蒙特卡洛法（Student-t，自由度5至7为标准选择）。审视三者之间的差异。
2. **报告 CVaR，而非 VaR。** 97.5% CVaR 对薄尾分布而言与99% VaR 保守程度相当，对厚尾分布则明显更为保守。巴塞尔采用97.5% CVaR；你也应该如此。
3. **始终进行压力测试。** 计算以下情景下的损失：1987年（单日-22%）、2008年（单日-9%，全年-38%）、2020年3月（单日-12%，单月-34%）、2022年（债券与股票同步回撤）。这些不是 VaR 事件；这些是*必须预案*的事件。
4. **依据尾部而非均值来确定仓位。** 构建杠铃型结构。大部分持仓使用正常波动性定制的防御性仓位；小规模高置信度的尾部对冲仓位使用损失有上限的工具（做多期权、深度现金缓冲、明确风险的垂直价差）。
5. **对头条数字打折扣。** 无论平台报告的99% VaR 是多少，**为了安睡，请在心里乘以二**。过于保守的代价是几个基点的机会成本。过于激进的代价是《华尔街日报》的讣告版面。

**陳馬的观点——VaR 是机构的表演秀，散户的解法在于结构，而非统计。** 我本人在亲历1998年、2008年和2020年3月的连番失败之后，对这些模型的理解是：VaR 的存在，主要是因为监管机构和风险委员会需要一个数字放在幻灯片上。这是表演秀。它在你最需要风险数字的机制下——波动率扩张的尾部、从压缩到扩张的机制转换、期权尾部驱动股票产生"不可能"走势的那些日子——系统性地低报风险。峰度在7至15之间而假设为3，这不是小的校准问题；这是模型形状根本错误。每个周期，大型银行都在眼泪中重新发现这一点，而应对措施总是给模型再加一个铃铛或口哨，而非承认整个模型类别是错的。

散户的答案不是更好的 VaR，而是更好的*投资组合形状*。我运行一个杠铃型结构，配置一个小型持续性的多波动率和尾部对冲仓位，其规模的设定使对冲账户最坏情形的损失等于我为对冲所支付的*已知*权利金——而非事后由 Student-t 告诉我的数字。对冲仓位的非对称性由结构所界定（做多期权、明确风险的垂直价差、深度现金缓冲），使最差一周的 CVaR 是我在仓位管理时主动选定的数字，而非事后由模型输出。这是一种反转：与其在一个毫无边界的投资组合上计算风险，不如选择一个最坏情形*结构性有界*的投资组合形状，让 VaR 数字充当理智检验，而非承重输入。

### 3. 常见误解

1. **"99% VaR 意味着最大损失以 VaR 为上限。"** 并非如此。99% VaR 是1%结果超过的阈值，对超过阈值的损失有多深只字不提。最坏情形的损失无界。
2. **"置信水平越高越保守。"** 对阈值本身是的，但对*估计质量*未必如此。用1,000个交易日计算的99.9%经验 VaR 只使用了字面上一个观测值。估计量的方差极大。更高的置信度不等于更多的知识。
3. **"如果参数法和历史法 VaR 一致，说明模型没问题。"** 两者通常在95%置信水平下一致，在99%以上开始分歧。平静时期会让分歧看起来很小，即便底层模型是错的。
4. **"VaR 是监管标准。"** 曾经是。巴塞尔协议III于2016年转向97.5%置信水平下的预期损失，并在2023年的FRTB框架中全面落地。如果你仍在引用原始 VaR，你已经落后超过七年。
5. **"蒙特卡洛跑100,000条路径比跑1,000条更准确。"** 只是*抽样噪声*更小。*模型误差*——即所选分布的错误程度——保持不变。从正态模型跑一百万条路径，给出的仍是正态尾部的答案。
6. **"分散投资总能降低 VaR。"** 对薄尾投资组合确实如此。对具有极端联合依赖（相关尾部事件）的重尾投资组合，VaR 可能违反次可加性——合并后的 VaR 可能超过各自 VaR 之和。CVaR 没有这一病态。
7. **"单日 VaR 用 $\sqrt{N}$ 缩放至N日 VaR。"** 仅在独立同分布正态收益率下成立。真实收益率存在波动性聚集和序列相关性。在压力时期，平方根规则会高估时间分散化收益。将其作为粗略参考，而非精确计算。
8. **"VaR 回测可以告诉你模型是否正确。"** 只是粗略地。一个模型在250天回测中可以出现正确数量的突破次数，却仍有严重错误的 CVaR——*阈值*匹配，但*尾部质量*不匹配。
9. **"压力测试是主观的，VaR 是客观的。"** 两者都是主观的。VaR 的主观性隐藏在分布选择和回溯窗口的选取中；压力测试的主观性则在情景设定中公开陈述。
10. **"CVaR 只是对 VaR 的小幅调整。"** 对正态分布而言确实如此，95%置信水平下比率约为1.25。对经验股票数据而言，CVaR 在99%置信水平下可达 VaR 的1.5倍——这个"小幅调整"，正是可承受损失与追加保证金之间的距离。

---

### 4. 问答环节

**问：我的券商显示单日95% VaR，我该重视吗？**
答：这是一个基准锚点——当它显示\$200时，你不应该对\$200的损失感到震惊。但不要止步于此。在脑海中对99%的估计翻倍（零售平台很少显示），对99.9%的估计乘以三，然后手动推演一个情景："如果标普500单日下跌10%会怎样？"最后这个数字才是影响你夜间安眠的唯一指标。

**问：为何巴塞尔协议III采用97.5%而非99%？**
答：两个原因。第一，在典型重尾股票分布下，97.5% CVaR 与99% VaR 的保守程度大致相当，监管约束力保持相近。第二，97.5%置信水平的 CVaR 估计量的抽样方差低于99%，因为它平均了更多观测值。这是保守性与统计稳定性之间的最佳平衡点。

**问：VaR 适用于期权投资组合吗？**
答：参数法 VaR 对期权投资组合效果极差，因为其盈亏相对于标的是非线性的。"Delta-Gamma"近似（线性加二次项）对小幅波动有一定帮助，但在尾部事件中会失效。蒙特卡洛法（正确应用时）是正确工具——模拟标的资产路径，在每条路径下对期权重新定价，然后排序盈亏。历史模拟法同样适用。

**问：历史法 VaR 的回溯窗口应多长？**
答：存在权衡。较短的窗口（250天）对当前机制响应更灵敏，但遗漏了较早的危机。较长的窗口（1,000至2,500天）涵盖更多危机，但混入了可能已不适用的旧机制。巴塞尔标准是250天非缩放窗口加上压力时期叠加层。从业者通常将500至750天作为默认值。

**问：时间根号缩放可靠吗？**
答：对均值回归序列（波动性、信用利差），它会高估N日风险。对趋势性序列（牛市中纯多头股票），它会低估风险。对独立同分布序列，它是精确的。真实序列在压力时期无一符合上述条件。将其作为粗略参考，而非精确计算。

**问：为何银行偏好单日 VaR，而养老金基金偏好年度 VaR？**
答：行动频率不同。银行可以在一天内调整账簿，按单日跨度管理。养老金基金的负债跨度为数十年，再平衡跨度为年度；单日 VaR 对一个无法像自营部门那样频繁交易的首席投资官而言，在操作层面毫无意义。

**问：VaR 与凯利准则有何关联？**
答：两者回答的是不同问题。凯利准则是*已知优势下的前瞻性仓位管理*。VaR 是*当前持仓的回顾性风险量化*。凯利规模的仓位与其预期损失和破产概率有已知关系；VaR 给出的是该损失分布的分位数。两者都应纳入你的工具箱。

**问：含有做空期权的投资组合，CVaR/VaR 比率会如何变化？**
答：CVaR/VaR 可能急剧上升。做空虚值看跌期权的预期损失有限（行权价较远时对 VaR 有利），但一旦触及行权价，尾部损失无界（对 CVaR 极为不利）。两个95% VaR 完全相同的策略，若一个做空尾部而另一个不是，99.9% CVaR 可能有天壤之别。这正是巴塞尔协议试图根除的"隐性尾部"类型。

**问：为何每当窗口内最差的一天滚出时，我的历史法 VaR 数字会跳变？**
答：因为历史法从字面上读取有限样本的分位数。当2020年4月从1,000天窗口中滚出时（约在2024年中期），99% VaR 会机械性地下降约30%——*即便投资组合本身没有任何变化*。这是方法本身的产物，而非真实风险的降低。使用年龄加权法或压力时期叠加层来平滑这一效应。

**问：有没有一个数字能将 VaR、CVaR 和压力测试综合为一？**
答：没有，而这正说明了问题所在。风险是多维的。一个投资组合同时存在正常日风险（波动性）、分位数风险（VaR）、平均尾部风险（CVaR）、最坏情形风险（压力测试）、尾部依赖风险（联合极端事件）以及流动性风险（你真的能出手吗？）。将这些压缩为一个数字，恰恰丢弃了在事情崩坏时最需要的那些信息。

**问：我在家里能计算的最简便"足够好的" VaR 是什么？**
答：取你的投资组合过去500个日度收益率，排序后读取第5百分位（99% VaR）和第25百分位（95% VaR）。计算最差5个收益率的平均值——这就是你的99% CVaR。你会比90%的专业 VaR 系统更保守，且只需要 Excel。

**小魚：** 听起来开场就已经在铺核心结论了。

**陳馬：** 确实如此。波动率尾部驱动全局。参数法 VaR 在95%置信水平下是有用的数字，在99%是猜测，在99.9%是虚构。这三句话之间的差距，吞噬了长期资本、2008年的结构性信贷部门，以及2020年3月的波动率目标基金。所以请认真听。

**[1:20——第一段：定义]**

**陳馬：** 先来认真讲定义。风险价值是**在给定概率和给定时间跨度内不会被超过的最大损失**。三个旋钮：置信水平——通常是95或99；时间跨度——通常是一天或十天；货币金额——VaR 数值本身。

**小魚：** 用一句话怎么说？

**陳馬：** "我们的单日99% VaR 是五百万美元。"这意味着在每100个交易日中的99天里，我们预期单日损失低于五百万。在第100天，损失可能更大——而 VaR 对*大到什么程度*保持沉默。最后这一句话，比今天要讲的其他任何内容都重要。

**[2:30——第二段：方法一，参数法]**

**小魚：** 方法一，参数法。

**陳馬：** 假设收益率服从正态分布——你统计课上那个钟形曲线。均值 μ，标准差 σ。那么 VaR 就是 $z_\alpha$ 乘以 σ 减去 μ，再乘以你的投资组合价值。Z 值：95%对应1.645，99%对应2.326，99.9%对应3.090。Excel 里就能算。

**小魚：** 优势？

**陳馬：** 快得像闪电。方差可加，相关性线性，一个持有一千个仓位的投资组合，电子表格就能搞定。

**小魚：** 劣势？

**陳馬：** 正态假设是*错的*。1987年10月19日，标普500单日暴跌20.5%。在参数模型下，这是一个19倍标准差事件，概率为 $10^{-79}$。意思是从现在到宇宙热寂都不应该发生。它发生了。就在一个星期一。

**小魚：** 所以95%置信水平下还行，99%下撒谎，99.9%下是科幻。

**陳馬：** 正是核心结论。

**[4:00——第三段：方法二，历史法]**

**陳馬：** 方法二，历史模拟法。完全抛弃正态假设。直接取你的投资组合过去一千天的盈亏数据，排序，读分位数。一千天中第10个最差的日子，就是你的99% VaR。没有模型，只有数据。

**小魚：** 优势？

**陳馬：** 肥尾免费附赠。如果2008年在你的窗口里，VaR 就包含了2008年。如果新冠在你的窗口里，VaR 就包含了新冠。数学不需要*猜测*尾部；它*记得*尾部。

**小魚：** 劣势？

**陳馬：** 你只能看到你见过的。如果你的窗口截止于2007年10月，数据里就没有2008年，雷曼倒下的那一刻，你的99% VaR 读起来就像95%水平。这正是2008年秋天许多银行部门的真实遭遇——他们的 VaR 是用2003至2007年平静期的数据校准的，所以突破"不可能"发生——直到真的发生了。

**小魚：** 展示三面板图。

**陳馬：** 左面板：1990年至2026年4月标普500日度收益率直方图。蓝色柱是经验分布，金色曲线是相同均值和标准差的正态分布。看那个肩部。正态曲线够不到数据实际所在的地方。经验直方图在-2%以下有更厚的肩部，左尾一直延伸到-10%甚至更远。正态曲线在那里基本上是零。

**[7:00——第四段：方法三，蒙特卡洛法]**

**陳馬：** 方法三，蒙特卡洛法。你写下一个模型，模拟一万或十万条投资组合路径，计算每条路径的盈亏，排序，读分位数。

**小魚：** 模型里放什么？

**陳馬：** 任何你能写下来的东西。可以是正态分布——这样蒙特卡洛结果与参数法一致，只多了抽样噪声。可以是自由度为5的 Student-t——尾部更厚，与数据更接近。可以是 GARCH——今日波动率取决于昨日。可以是跳跃扩散——正态噪声加偶发性泊松跳跃，对应崩盘日。可以是机制转换的高斯混合模型。

**小魚：** 优势？

**陳馬：** 完全灵活。可以捕捉非线性收益——期权、结构化产品。可以捕捉路径依赖——障碍产品、可赎回债券。你想构建什么世界就构建什么世界。

**小魚：** 劣势？

**陳馬：** 你构建什么世界就是什么世界。垃圾进，垃圾出。如果模型在尾部有误，蒙特卡洛 VaR 在尾部就有误——还带着精确性的幻觉。用错误模型跑十万条路径，给你的只是一个信心满满的错误答案。

**[9:00——第五段：CVaR / 预期损失]**

**陳馬：** 现在来看那个更好的数字。CVaR，条件风险价值，也叫预期损失。

**小魚：** 定义？

**陳馬：** *既然损失已经超过 VaR 阈值，平均损失是多少？* 你先算出 VaR，然后对所有突破 VaR 的结果取平均。这就是 CVaR。

**小魚：** 为什么它更好？

**陳馬：** 两个原因。第一，它回答了 VaR 所回避的问题——突破有多严重。第二，它具有数学意义上的*一致性*。VaR 对重尾分布可能违反次可加性——合并两个投资组合后的 VaR 可能大于两者之和。这在数学上是病态的，会激励隐性尾部风险集中。CVaR 没有这个问题。

**小魚：** 监管机构转向 CVaR 了吗？

**陳馬：** 是的。巴塞尔协议III在2016年《交易账簿基本审查》中，以97.5%置信水平的预期损失取代了 VaR。大型银行已经切换到 CVaR 好几年了。零售平台大多仍在引用 VaR——作为基准没问题，但如果你的平台显示 CVaR，务必参考它。

**陳馬：** 展示峰度图表。

**小魚：** 这是1990年以来标普500日度收益率的滚动五年峰度。

**陳馬：** 注意那条水平虚线，标注在3的位置。那是收益率为真正正态分布时峰度的数值。注意经验折线*高于3*——在样本的每一天都是如此。三十六年来，它从未触碰过正态分布的理论值。大多数时间介于7至12之间。包含1987年的窗口超过30。2008年的窗口约为12。即便是2003至2007年这样平静的窗口也超过5。

**[13:00——第七段：实用框架]**

**陳馬：** 那么实际应该怎么做？

**小魚：** 带我们过一遍。

**陳馬：** 五步。第一：同时运行三种方法——参数法、历史法、Student-t 蒙特卡洛法。审视三者之间的差异。第二：引用97.5% CVaR 而非99% VaR，监管层面的保守程度相当，数学上更诚实。第三：始终进行明确的压力测试——1987年、2008年、2020年3月、2022年。这些不是 VaR 事件，这些是*必须预案*的事件。第四：依据尾部而非均值定仓——杠铃型结构：大部分持仓用正常波动性定制的防御性仓位，小仓位做明确风险边界的对冲结构。第五：对头条数字打折扣——无论平台报告的99% VaR 是多少，*为了安睡，请乘以二*。过于保守的代价是几个基点，过于激进的代价是讣告版面。

**[14:30——第八段：互动实验]**

**小魚：** 互动演示环节。

**陳馬：** 本周的实验是一个 VaR 计算器。三个滑块：投资组合价值从一万到一千万美元；波动性假设从年化5%到40%；置信水平——90、95、99、99.5、99.9。还有一个方法切换开关：正态分布、Student-t（带自由度滑块），或基于过去五年标普数据重新抽样的历史法。

**[16:30——结尾]**

**小魚：** 总结一下。

**陳馬：** 三个要点。第一：VaR 是一个*阈值*，CVaR 是*平均突破幅度*——有条件的话，始终引用 CVaR。第二：三种方法在尾部全都失效——真实股票数据的峰度在7至15之间，从未是3，越深入尾部，三种方法分歧越大。第三：正确的风险系统同时运行三种方法加明确的压力测试，并将三者之间的差异视为数字中不可消除的不确定性。波动率尾部驱动全局，模型不是世界，世界终将超出模型的预期——请在它发生之前做好规划。

**小魚：** 更深层的那个质疑呢？

**陳馬：** 就我自己的账户而言，VaR 是机构的表演秀。它的存在，是因为风险委员会需要一个数字放在幻灯片上，而它在你最需要风险数字的机制下——波动率扩张的尾部、期权尾部驱动股票走出"不可能"走势的那些日子——系统性地低报。散户的解法不是更好的模型，而是更好的投资组合形状。构建一个杠铃型结构，配置一个小型持续性尾部对冲仓位，规模的设定使对冲账户的最坏情形损失是你为对冲所支付的*已知*权利金——而非事后由 Student-t 输出的数字。选择一个最坏情形结构性有界的形状，让 VaR 数字充当理智检验，而非承重输入。
